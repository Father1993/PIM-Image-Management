# Инструкция для тестирования в Postman

## Шаг 1: Получение токена

**Метод:** `POST`  
**URL:** `{{PIM_API_URL}}/sign-in/`

**Headers:**
```
Content-Type: application/json
```

**Body (raw JSON):**
```json
{
  "login": "ваш_логин",
  "password": "ваш_пароль",
  "remember": true
}
```

**Ответ:** Скопируйте токен из `data.access.token`

---

## Шаг 2: Создание товара

**Метод:** `POST`  
**URL:** `{{PIM_API_URL}}/product/`

**Headers:**
```
Authorization: Bearer {{ваш_токен}}
Content-Type: application/json
```

**Body (raw JSON):** Используйте пример из `postman_example_request.json`

---

## Важные замечания

1. **Замените переменные:**
   - `{{PIM_API_URL}}` - URL вашего PIM API (например: `https://demodata.compo-soft.ru/api/v1`)
   - `{{ваш_токен}}` - токен из шага 1

2. **Проверьте catalogId:** Убедитесь, что категория с ID существует в системе

3. **Проверьте структуру catalog:** Объект `catalog` должен содержать правильные данные категории

4. **Ошибка 500:** Может быть связана с:
   - Неправильной структурой данных
   - Отсутствием обязательных полей
   - Проблемами с категорией
   - Валидацией данных на стороне сервера
